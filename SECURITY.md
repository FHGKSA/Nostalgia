| バージョン | サポート状況 |
| ------- | ---------- |
| 1.0.x   | ✅ サポート |

## 脆弱性の報告

プロジェクトでセキュリティ脆弱性を発見した場合は、以下の方法で報告してください：

### 報告方法

1. **GitHub Security Advisory** を使用（推奨）
   - [Security Advisories](https://github.com/FHGKSA/Nostalgia/security/advisories)
   - 非公開で報告が可能

2. **メール報告**
   - セキュリティ問題は `security@yourdomain.com` まで
   - 件名に「[Nostalgia] Security Issue」を含める

### 報告内容

以下の情報を含めてください：

- 脆弱性の詳細説明
- 再現手順
- 影響範囲・重要度
- 可能であれば修正案

### 対応プロセス

1. **確認** - 24時間以内に受信確認
2. **調査** - 72時間以内に初回レスポンス
3. **修正** - 重要度に応じて1-14日で修正版リリース
4. **公表** - 修正後、適切なタイミングでアドバイザリ公開

## セキュリティ上の注意事項

### ユーザー向け

- **Ollama API**: 信頼できるモデルのみ使用
- **設定ファイル**: 機密情報（APIキー等）を含めない
- **ログファイル**: 個人情報が含まれる可能性があるため共有注意
- **アセットファイル**: 著作権や個人情報を含む画像・音声の使用注意

### 開発者向け

- **依存関係**: 定期的な脆弱性スキャン実施
- **入力検証**: ユーザー入力の適切なサニタイゼーション
- **ログ出力**: 機密情報のログ出力防止
- **実行権限**: 最小権限の原則

## 既知の制限事項

- Ollama APIへのアクセスはローカルネットワークのみ想定
- ファイルアクセスはプロジェクトディレクトリ内のみ
- 外部ネットワーク通信はOllama APIのみ

**注意**: この情報は一般的なセキュリティ慣行に基づいています。具体的なセキュリティ要件は使用環境に応じて調整してください。
